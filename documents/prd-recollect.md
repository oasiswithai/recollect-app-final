# PRD: Recollect - AI-Powered Knowledge Card Management System

## 1. Introduction/Overview

**Recollect**은 흩어진 스크린샷, 링크, 이미지 정보를 한 곳에 수집하고, AI가 이를 자동으로 분석·태깅·분류하여 '찾아볼 수 있는 지식 카드'로 통합 관리하는 웹 애플리케이션입니다.

### Problem Statement
사용자들은 웹 서핑 중 유용한 정보를 스크린샷, 북마크, 이미지 등 다양한 형태로 저장하지만:
- 저장한 정보를 나중에 찾기 어려움
- 수동으로 정리하고 태깅하는 데 시간이 많이 소요됨
- 관련된 정보들 간의 연결성을 파악하기 어려움
- 여러 플랫폼에 흩어져 있어 통합 관리가 불가능함

### Solution
AI 기반 자동 분석 및 카드형 UI를 통해 정보 수집부터 재검색까지의 전 과정을 자동화하고 시각화하여, 사용자가 지식을 체계적으로 관리하고 쉽게 다시 찾을 수 있게 합니다.

## 2. Goals

1. **자동화를 통한 시간 절약**: AI가 콘텐츠를 자동으로 분석·태깅·분류하여 사용자의 수동 작업 최소화
2. **높은 재검색률**: 사용자가 저장한 정보의 70% 이상을 다시 찾아볼 수 있도록 검색/탐색 경험 최적화
3. **지식 연결성 강화**: AI가 유사한 콘텐츠를 자동으로 연결하여 지식 체계화
4. **사용자 만족도 향상**: 직관적인 카드 기반 UI와 다크/라이트 모드 지원으로 사용성 극대화
5. **지속적인 사용 유도**: 주 2회 이상 재방문하는 활성 사용자 확보

## 3. User Stories

### Primary Users: 일반 웹 서핑 사용자
- **As a** 일반 사용자, **I want to** 마음에 드는 이미지나 링크를 업로드, 드래그앤드롭으로 쉽게 저장하고 **so that** 나중에 다시 찾아볼 수 있다
- **As a** 일반 사용자, **I want to** 검색창에 키워드만 입력하면 관련 카드를 찾을 수 있기를 **so that** 폴더를 일일이 뒤지지 않아도 된다
- **As a** 모바일 사용자, **I want to** 모바일에서도 데스크탑과 동일한 경험을 **so that** 언제 어디서나 내 지식 카드에 접근할 수 있다

## 4. Functional Requirements

### FR1: 데이터 수집 기능
1. 사용자는 웹 인터페이스에서 드래그앤드롭으로 이미지/링크/텍스트를 업로드할 수 있어야 한다
2. 사용자는 직접 파일을 선택하여 업로드할 수 있어야 한다
3. 시스템은 업로드된 콘텐츠의 메타데이터(출처 URL, 업로드 시각, 타이틀 등)를 자동으로 추출해야 한다
4. 시스템은 이미지, 링크, 텍스트를 모두 지원해야 한다

### FR2: AI 기반 지식 카드 생성
5. 시스템은 업로드된 콘텐츠를 AI로 분석하여 핵심 내용을 요약해야 한다
6. 시스템은 콘텐츠에 자동으로 태그를 생성하고 주제별로 분류해야 한다
7. 시스템은 이미지에서 OCR을 통해 텍스트를 추출하고 키워드를 분석해야 한다
8. 시스템은 비슷한 내용의 기존 카드를 찾아 연결 제안을 해야 한다
9. 시스템은 분석 결과를 카드 형태의 UI로 표시해야 한다
10. 각 카드는 썸네일, 제목, 요약, 태그, 출처, 날짜 정보를 포함해야 한다

### FR3: 지식 카드 뷰어 & 검색
11. 메인 화면은 **Pinterest 스타일 Masonry 레이아웃**으로 카드를 표시해야 한다 (가변 높이의 카드가 자연스럽게 배치)
12. **메인 화면 상단에 필터 탭/칩**이 고정되어 있어야 한다 (자주 쓰는 태그, 최근 태그, AI 추천 태그)
13. 사용자는 필터 탭을 클릭하여 **해당 태그의 카드만 즉시 필터링**할 수 있어야 한다
14. 사용자는 검색창에 키워드를 입력하여 관련 카드를 찾을 수 있어야 한다
15. 사용자는 태그, 출처, 날짜로 카드를 필터링할 수 있어야 한다 (사이드바 또는 드롭다운)
16. 사용자는 카드를 클릭하여 세부 정보, AI 요약, 관련 카드 연결을 볼 수 있어야 한다
17. 시스템은 사용자의 검색 패턴을 학습하여 맞춤 추천을 제공해야 한다
18. 검색 결과는 관련성 순으로 정렬되어야 한다
19. **필터 탭은 가로 스크롤 가능**해야 하며, 모바일에서도 접근성이 좋아야 한다

### FR4: AI 기반 콜렉션 카드 자동 생성
20. 시스템은 **태그 기반으로 콜렉션을 자동 생성**해야 한다 (동일 태그를 가진 카드들을 그룹화)
21. 시스템은 유사한 주제/출처/시간대를 기반으로 카드를 자동 그룹화해야 한다
22. AI는 **사용자가 자주 보는 태그 조합**을 학습하여 맞춤 콜렉션을 제안해야 한다
23. 시스템은 자동 생성된 콜렉션을 사용자에게 제안해야 한다
24. 사용자는 제안된 콜렉션을 수락/거절/수정할 수 있어야 한다
25. 사용자는 수동으로 콜렉션을 생성하고 카드를 추가할 수 있어야 한다
26. 콜렉션 카드는 포함된 카드 수, 대표 이미지, 자동 생성된 제목, **주요 태그**를 가져야 한다
27. **콜렉션 카드도 메인 화면 Masonry 레이아웃에 함께 표시**되어야 한다 (일반 카드와 혼합)

### FR5: 반응형 UI 최적화
28. UI는 모바일(320px~)과 데스크탑(1024px~) 모두에서 최적화되어야 한다
29. 시스템은 다크 모드와 라이트 모드를 지원해야 한다
30. 모바일에서는 Masonry 카드 레이아웃이 1-2열로 자동 조정되어야 한다
31. 세부 정보는 모달/슬라이딩 패널로 표시되어야 한다
32. 시스템은 사용자의 시스템 설정에 따라 자동으로 테마를 적용해야 한다
33. **필터 탭은 모바일에서 고정 헤더**로 유지되어야 한다 (스크롤 시에도 보임)

## 5. Non-Goals (Out of Scope)

이번 버전에서는 다음 기능들을 **포함하지 않습니다**:

1. **협업 기능**: 다른 사용자와 카드 공유, 댓글, 공동 편집 등
2. **소셜 기능**: 팔로우, 피드, 좋아요 등의 소셜 네트워크 요소
3. **오프라인 지원**: 네트워크 없이 작동하는 오프라인 모드
4. **PDF/문서 파일 분석**: 이미지, 링크, 텍스트 외의 파일 형식
5. **음성/비디오 콘텐츠**: 오디오나 영상 파일의 수집 및 분석
6. **브라우저 히스토리 자동 수집**: 명시적 수집 외의 자동 추적
7. **결제/유료 플랜**: 무료 서비스로 시작하며 수익화 모델은 추후 고려
8. **써드파티 통합**: Notion, Evernote 등 외부 서비스와의 연동

## 6. Design Considerations

### UI/UX Requirements
- **카드 기반 디자인**: Pinterest 스타일의 **Masonry 레이아웃** (가변 높이 카드)
- **태그 중심 네비게이션**: 메인 화면 상단에 필터 탭/칩을 배치하여 빠른 태그 필터링
- **시각적 명료성**: 한눈에 정보를 파악할 수 있는 깔끔한 디자인
- **맥락 중심**: 카드 간 연결이 시각적으로 명확하게 표현
- **최소 인터랙션**: 클릭 수를 최소화한 직관적인 UX (필터 탭 원클릭)
- **자동화 강조**: AI가 작동하는 과정을 시각적으로 피드백
- **혼합 레이아웃**: 일반 카드와 콜렉션 카드가 자연스럽게 섞여 표시

### Color Scheme & Theme
- **라이트 모드**: 화이트/라이트 그레이 베이스, 파스텔 액센트
- **다크 모드**: 다크 그레이/블랙 베이스, 비비드 액센트
- **자동 전환**: 시스템 설정 또는 사용자 선택에 따라 전환

### Key Components
- **Card Component**: 썸네일, 제목, 요약, 태그 칩, 메타데이터 (가변 높이)
- **Collection Card**: 그룹화된 카드들의 그리드 미리보기, 주요 태그 표시
- **Filter Tab Bar**: 가로 스크롤 가능한 태그 칩 리스트 (상단 고정)
  - 자주 쓰는 태그 (빈도 기반)
  - 최근 사용한 태그
  - AI 추천 태그 (현재 보고 있는 내용 기반)
- **Search Bar**: 자동완성 및 고급 필터 통합
- **Detail Modal**: 카드 세부 정보 및 관련 카드 표시
- **Upload Zone**: 드래그앤드롭 인터랙션 (플로팅 버튼 또는 헤더)

## 7. Technical Considerations

### Frontend
- **Framework**: React/Next.js (SSR/SSG 지원)
- **State Management**: Context API 또는 Zustand
- **Styling**: Tailwind CSS (다크 모드, 반응형 쉽게 구현)
- **Masonry Layout**: `react-masonry-css` 또는 `masonic` 라이브러리
- **Image Handling**: Next/Image 최적화 활용
- **Infinite Scroll**: `react-infinite-scroll-component` 또는 Intersection Observer API

### Backend & AI
- **API**: RESTful API 또는 GraphQL
- **AI/ML Model**:
  - 텍스트 분석: GPT-4 API 또는 Claude API
  - OCR: Tesseract.js 또는 Google Cloud Vision API
  - 이미지 분석: CLIP 모델 또는 Vision API
- **Vector Database**: 유사도 검색을 위한 Pinecone/Weaviate
- **메타데이터 추출**: Open Graph Protocol, Metadata scraping

### Storage & Database
- **File Storage**: AWS S3 또는 Cloudinary (이미지 최적화)
- **Database**: PostgreSQL (관계형 데이터) + Vector DB (임베딩)
- **Caching**: Redis (검색 결과, 추천 캐싱)

### Browser Extension
- **Target**: Chrome/Edge (Manifest V3)
- **기능**: 현재 페이지 캡처, 선택 영역 캡처, 링크 저장

### Performance
- **목표**: 페이지 로드 < 2초, AI 분석 완료 < 5초
- **Lazy Loading**: 카드 무한 스크롤 시 점진적 로딩
- **CDN**: 정적 파일 및 이미지 전송 최적화

## 8. Success Metrics

### Primary Metrics
1. **재검색률 (Retrieval Rate)**: 저장한 콘텐츠를 다시 검색/조회하는 비율 **≥ 70%**
2. **사용자 만족도**: NPS 또는 만족도 설문 점수 **≥ 8/10**
3. **재방문율**: 주 2회 이상 방문하는 활성 사용자 비율 **≥ 40%**

### Secondary Metrics
4. **수집된 콘텐츠 수**: 사용자당 평균 저장 카드 수 **≥ 50개/월**
5. **AI 태깅 정확도**: 사용자가 AI 생성 태그를 수정하지 않는 비율 **≥ 80%**
6. **검색 성공률**: 검색 후 카드를 클릭하는 비율 **≥ 60%**
7. **콜렉션 수락률**: AI 제안 콜렉션을 수락하는 비율 **≥ 50%**

### User Engagement Metrics
8. **평균 세션 시간**: 5분 이상
9. **모바일 vs 데스크탑 비율**: 모바일 40%, 데스크탑 60%
10. **확장 프로그램 사용률**: 전체 업로드 중 확장 프로그램 비율 **≥ 30%**
11. **필터 탭 사용률**: 검색 대비 필터 탭 클릭 비율 **≥ 60%** (태그 중심 탐색 선호도)

### Measurement Methods
- **Analytics**: Google Analytics 또는 Mixpanel
- **User Feedback**: 인앱 설문, NPS 조사 (월 1회)
- **A/B Testing**: 검색 알고리즘, UI 변경 등에 대한 테스트
- **Logging**: 검색 쿼리, 클릭 패턴, AI 제안 수락/거절 로깅

## 9. Open Questions

1. **AI 모델 선택**: 비용 대비 성능을 고려했을 때 GPT-4 vs Claude vs 오픈소스 모델 중 무엇을 사용할 것인가?
2. **데이터 프라이버시**: 사용자 콘텐츠를 서버에 얼마나 저장할 것인가? 개인정보 보호 수준은?
3. **확장 프로그램 우선순위**: Chrome 확장 프로그램을 MVP에 포함할 것인가, 아니면 2단계로?
4. **AI 분석 시간**: 실시간 분석 vs 백그라운드 처리? 사용자 경험에 미치는 영향은?
5. **유사도 임계값**: 관련 카드 제안 시 몇 %의 유사도부터 표시할 것인가?
6. **저장 용량 제한**: 무료 사용자에게 허용할 저장 용량은? (이미지 최적화 수준 결정 필요)
7. **다국어 지원**: 초기 버전에서 영어/한국어만 지원? 추후 확장 계획은?
8. **콜렉션 생성 빈도**: AI가 얼마나 자주 새 콜렉션을 제안할 것인가? (스팸처럼 느껴지지 않도록)
